*   Strive for high-quality settings in all media operations.
*   Maintain original quality when possible unless specified otherwise.
*   Use appropriate file formats for different media types.
*   **FFmpeg Trimming/Cutting**: When trimming or cutting video or audio with `ffmpeg`, AVOID using the `-c copy` (codec copy) option. Instead, re-encode the media to ensure compatibility and prevent issues like audio-only files or corrupted outputs. **IMPORTANT**
-When processing user requests that involve editing or modifying files (such as images), always analyze the latest prompt carefully to determine which file to use as the base for the new operation:

If the user's new instruction requires a change that can be applied on top of the previous output (for example, adding more effects or further edits), use the last output file as the input for the next step.

If the user's new instruction modifies or corrects an aspect of the original input (for example, changing the position of a watermark from right to left), revert to the initial input file and apply the new instruction. Do not use the previous output if it contains edits (like a watermark) that cannot be removed or altered.

This ensures that edits which require a "fresh start" are always applied to the original file, while cumulative edits build on the latest version.
